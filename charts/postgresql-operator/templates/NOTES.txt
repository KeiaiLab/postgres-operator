📦 Operator Manager:
  kubectl get deployment -n {{ .Release.Namespace }} {{ include "postgresql-operator.fullname" . }}-controller-manager

📋 ServiceAccount + RBAC:
  kubectl get sa,clusterrole,clusterrolebinding -n {{ .Release.Namespace }} -l app.kubernetes.io/instance={{ .Release.Name }}

🧬 CRDs (PostgresCluster, BackupJob):
  kubectl get crd postgresclusters.postgres.keiailab.io
  kubectl get crd backupjobs.postgres.keiailab.io

CRD lifecycle을 별도로 관리하려면 설치 시 Helm 기본 옵션을 사용:
  helm install {{ .Release.Name }} keiailab/postgresql-operator --skip-crds

{{- end }}

{{ if eq .Values.replicas 1.0 -}}
⚠️  replicas=1 — alpha default. For HA (P2-M1 leader election), set replicas=2+:
  helm upgrade {{ .Release.Name }} keiailab/postgresql-operator --set replicas=3
{{- end }}

{{ if not .Values.citusLibPQ.dsn -}}
ℹ️  CITUS_LIBPQ_DSN not set — using NullExecutor (P11-M0 spike default).
   pg_dist_node SQL는 적용되지 않으며, desired state는 Status.Topology에만 표면화됨.
   실 SQL apply를 활성화하려면 (P0-6 phase 2a):
     helm upgrade {{ .Release.Name }} keiailab/postgresql-operator \
       --set citusLibPQ.dsn="host=<coord-svc-dns> port=5432 user=postgres dbname=postgres sslmode=require"
{{- else -}}
✅ LibPQExecutor enabled (P0-6 phase 2a) — pg_dist_node 변경이 실 PG에 적용됨 (single-cluster mode).
{{- end }}
